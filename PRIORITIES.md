## 2026-06-30 afternoon pass ‚Äî Per-entry-price paper performance breakdown (+ maintenance recovery)

**Maintenance recovery (this run started on a broken tree ‚Äî `cargo check` had 6 errors).** The previous session left `src-tauri/src/paper/mod.rs` with a classic `patch`-tool template-literal corruption: a single 4,179-character line containing 109 literal `\n` escape sequences (instead of real newlines) that was a corrupted copy of the `PaperEntryPriceStats` struct, its impl, and a duplicate `compute_entry_price_stats` function. The corruption made `PaperEntryPriceStats` undeclared at the call site of `compute_entry_price_stats`, producing 6 `E0425` / `E0433` errors. Two real bugs were also embedded in the freshly-added code:

1. `BTreeMap<f64, PaperEntryPriceStats>` does not compile ‚Äî `f64` does not implement `Ord`, only `PartialOrd`. `BTreeMap::entry` requires `K: Ord`. Re-keyed to `BTreeMap<usize, PaperEntryPriceStats>` keyed by bucket index (0..4). The 5 canonical buckets stay readable as static `&[(f64, f64, &str)]`; the index is the map key, so iteration order is naturally sorted ASC.
2. The duplicated struct definition was on the same corrupted line ‚Äî once the line was deleted, a clean `PaperEntryPriceStats` struct + `impl` block was re-inserted between `compute_entry_price_stats` and the existing `SessionDelta` doc comment.

Surgical fix: a 100-line Python script (`hermes-fix-papermod.py`) opened the file, line-based deleted the corrupted 4,179-char line (index 467, 0-based), and spliced a clean struct + impl block in. Then the `BTreeMap<f64, ‚Ä¶>` ‚Üí `BTreeMap<usize, ‚Ä¶>` substitution was applied with `patch`. Also fixed two cosmetic over-indentations introduced by the previous session: (a) `PaperAnalytics` interface in `src-ui/src/types/prizepicks.ts` had 4 lines with 4 extra leading spaces; (b) 6 render lines in `PrizePicksPredictionsPanel.tsx` were indented at 12 spaces when their siblings were at 6 spaces. Both fixed.

- `src-tauri/src/analysis/kelly_shrinkage.rs` ‚Äî new module. `compute_shrinkage(&[ResolvedForBrier])` returns a `KellyShrinkageReport { multiplier, n, brier, base_rate, climatology_brier, brier_skill_score, sample_factor, calibration_factor, reason }`. Cold start (n=0) returns multiplier=1.0. Cold but non-zero (1 ‚â§ n < 30) fades linearly from 0.50 ‚Üí 1.0 via `sample_factor`. Climatology Brier = `base_rate * (1 - base_rate)` (binary). 10 unit tests cover cold start, single prediction, small sample, warm near-climatology, sharp well-calibrated (BSS=1), mildly miscalibrated (BSS<0), overconfident (floored at MIN_MULT=0.50), degenerate all-wins (no NaN), parse_hit_outcome strings, and the predictions adapter.
